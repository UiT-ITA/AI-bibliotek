## Tekster er ment for å kunne: 
- limes inn som instruksjoner til Github Copilot, Claude, med mer.  
- legges inn i filen .github/copilot-instructions.md
- spesifike .github/instructions/{meningsfullt navn}/{formål}.instructions.md.

### Referanser
- [Github docs, add-repository-instructions](https://docs.github.com/en/copilot/how-tos/copilot-on-github/customize-copilot/add-custom-instructions/add-repository-instructions)

## Retningslinjer: 
- Det vil KANSKJE komme en mal for skills, etc.
- Skriv en header, med informasjon til brukeren. Dette kan være:
  - Overordnet kort forklaring
  - Råd / advarsler
  - Tilpasninger som må/bør gjøres før bruk
- Filen trenger ikke å være komplett, med alle steg
- Hvis det er en skisse, skriv det gjerne øverst.
- Og ja, jeg vurderer en skill, som sjekker at postingene til dette repo'et følger "guidelines".
